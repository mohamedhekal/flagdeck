# Contributing

Thanks for considering a contribution.

## Development setup

```bash
git clone <repo>
cd flagdeck
composer install
composer test
```

## Guidelines

- Prefer small, focused pull requests.
- Add or update tests for behavior changes.
- Run `composer format` (Pint) before opening a PR.
- Do not expand scope with unfinished Filament/UI work unless the issue asks for it.
- Document user-facing changes in `CHANGELOG.md`.

## Branching

- `main` — stable
- `feat/*`, `fix/*`, `docs/*` — work branches

## Pull requests

- Describe the problem and the approach.
- Link related issues.
- Ensure CI is green.

## Good first issues

Look for labels `good first issue` and `help wanted` once the GitHub repo is published.
